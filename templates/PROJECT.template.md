## Reglas

### Código
1. Seguir estilo existente del proyecto
2. Funciones pequeñas y enfocadas
3. Nombres descriptivos en inglés
4. Comentarios solo cuando son necesarios

### Commits
Formato convencional:
- `feat(scope): nueva funcionalidad`
- `fix(scope): corrección de bug`
- `refactor(scope): refactorización`
- `docs: documentación`

### Prohibido
- Crear archivos de documentación no solicitados
- Agregar dependencias sin justificación
- Comentarios obvios o redundantes
- Código muerto o no utilizado

## Estructura
```
src/        - Código fuente principal
lib/        - Librerías y utilidades
tests/      - Tests
scripts/    - Scripts auxiliares
```

## Comandos útiles
